Project Idea

A huge part of Twitter conversation revolves around news and politics. That makes it an excellent place to measure public opinion, especially during election campaigns. Twitter Sentiment Analysis can provide interesting insights on how people feel about a specific candidate (and you could even track sentiment over time to see how it evolves).

We are using machine learning to find out! For the past few days, team have been collecting tens and thousand's of tweets posted by users from around the world that discuss this topic and classifying them using sentiment analysis . That is, for each tweet published that mentions either Donald Trump or Joe Biden, tags it with its sentiment, which can be either positive, neutral, or negative.
Election day looms closer and closer every week. US Politics are rapidly becoming the preferred conversation topic for millions of Americans and non-Americans worldwide. What are these people saying? What do they think? What are their opinions? How do they feel?

Having millions and millions of tweets that are talking about Donald Trump and Joe Boden is somehow useless if we can’t find an easy way to analyze the data. Just manually looking at the raw tweets doesn’t scale, there’s too many of them. For this purpose, we created a ML model to analyze all of these tweets and get their sentiment and extract the most relevant keywords.

Tasks

Extract data
Twitter API and Twitter data streaming methods were used extract tweet data. We used an hour window every day for seven days to extract tweet data.
Data cleaning and normalization
From our raw data inputs we cleaned and normalized the data with the goal of extracting useful keywords and predict the tweet sentiment.
Database management
Normalized data was then loaded into a Postgres development database prior to being converted into a Heroku database.
Analysis and visualization
Visualizations were built using variety of JS libraries.
Build dashboard and website
The dashboard function of the website was built around the interactive visualizations.

Limitations

Data Volume
Volume of tweets per minute is huge. Due to limitation of hardware, storage and practicality on volume of Data, team decided to retrieve cloe to 10K tweets per day for a week duration.
Data Retrieval
Twitter allows data retrieval through API up to seven days time period.
Word Classification
Too many techniques are available to classify tweet though none shows accuracy to classify tweets correctly over 80%.
EDA Process
Extensive EDA process is involved that includes spell check for every word.
Data Processing Resources
Memory and CPU intensive data processing.
